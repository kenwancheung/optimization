Lecture 03: Mixed-Integer Programming¶

An integer linear program (IP) is a linear program where the decision variables are restricted to integer values. A mixed integer linear program (MIP) is a linear program where some of the decision variables are restricted to integer values.

MIPs are useful in many applications. Typically MIPs can be applied to solve problems that involve discrete choices. For example, a manager may need to choose between several projects subject to a budget constraint. The choice is binary, either choose or reject a project. MIPs can also be used to solve linear programs with logical conditions. For example, a corporation needs to decide between office locations, but must have at least one office in each region.

The most frequently used method to solve a mixed-integer program is the branch and bound method. There are other methods available including the cutting planes and branch and cut methods. This lecture discusses the branch and bound method to solve mixed integer programs.

Mixed Integer Linear Program¶

This section begins by defining a mixed-integer linear program:

minxs.t.cTx+hTyAx+By≤bx∈Zn+y∈Rp+minxcTx+hTys.t.Ax+By≤bx∈Z+ny∈R+p

where xx and cc are n×1n×1 vectors, yy and hh are p×1p×1 vectors; AA is a m×nm×n matrix, BB is a m×pm×p matrix, bb is a m×1m×1 vector. The components of the vector xx are nonnegative integers and the components of the vector yy are nonnegative real numbers.

There are several useful types of logical constraints. Some of the most frequently used are presented below:

Description Equation
If xixi is selected, then xjxj must be selected xi−xj≤0xi−xj≤0
xixi or xjxj must be selected, but not both xi+xj≤1xi+xj≤1
If xixi is selected, then xjxj must not be selected xi−xj≤1xi−xj≤1

maxx34x1+x234x1+x2≤32x1+x2≤6−x1+x2≤1x1,x2∈Z+maxx34x1+x234x1+x2≤32x1+x2≤6−x1+x2≤1x1,x2∈Z+

Search method¶

As in the linear program, we can try to find the solution to an integer program by a search method. The problem is that the number of calculations and time needed to search for the solution increases dramatically as the number of variables and conditions increase. Nonetheless, it is instructive to work through a manageble example, and then contrast with the branch and bound method.

Let's look at an application of the search method to a integer program. Suppose a manager has to decide between 4 projects with cost and value given by:

Project Cost Value
1 8 25
2 2 2
3 6 18
4 5 6

The manager can choose one or more projects as long as the cost is equal to or less than 1010. The goal of the manager is choose among the projects in order to maximize value.

The maximization problem can be written as:

maxx25x1+2x2+18x3+6x48x1+2x2+6x3+5x4≤10x1,x2x3x4∈{0,1}maxx25x1+2x2+18x3+6x48x1+2x2+6x3+5x4≤10x1,x2x3x4∈{0,1}

Notice that xi=1xi=1 implies that the manager chose project ii and xi=0xi=0 implies that the manager rejected project ii.

A naive search method enumerates all possible combinations of the values that x1x1, x2x2, x3x3 and x4x4 can take, checks the feasibility conditions and evaluates the objective function at every enumeration. In this case, there are 42=1642=16 possible combinations. All combinations of x1x1, x2x2, x3x3 and x4x4 with their associated objective function value are shown below (a value of −9999−9999 is assigned to combinations that are infeasible).

The table below shows all possible combinations of x1x1, x2x2, x3x3, and x4x4 together with their respective objective function value. The results are sorted by the value of the objective function from the highest to the lowest. All infeasible combinations are assigned a objective function value of −9999−9999. The search method finds that the manager should choose projects 1 and 2 and reject projects 3 and 4 to maximize value.

Branch and bound¶

The branch and bound method is a technique to solve a mixed-integer program. It is a helpful method because enumerating all possible solutions is a time consuming process. A complete enumeration identifies all possible values for the decision variables. For example, if there are 1010 binary variables, then there will be 210210 enumerations. The branch and bound method starts by enumerating potential solutions but it eliminates enumerations as much as possible, improving the time to find the optimal solution.

The branch and bound method begins by solving a LP relaxation of the original problem. The LP relaxation involves solving the mixed integer program without the integrality restrictions. It then creates two separate subproblems based on restricting a particular control variable to be greater than a given integer threshold for the first subproblem and less than another threshold for the second subproblem (branching). The thresholds are set as the rounded up and down of the solution to the control variable from the LP relaxation problem.

Each subproblem is solved based on a linear program technique (i.e., LP relaxation of the subproblem). Each subproblem is evaluated as shown below:

  • If the subproblem has no feasible solution, then it is eliminated;

  • If the subproblem has an integer optimal solution, then we compare it to the existing optimal solution (incumbent).

    • If the new solution improves the value of the objective function from the incumbent solution, then we update the incumbent solution based on the new solution.
    • Else we don't update the incumbent solution.
  • If the solution to the subproblem is not all integers but has a better objective function than the incumbent solution, then we further subdivided the subproblem, repeating the entire procedure described above.

A lower (upper) bound to a mixed-integer linear minimization (maximization) problem is the value of the objective function at a solution that ignores the integrality restrictions. An upper (lower) bound to the mixed-integer linear minimization (maximization) problem is the value of the objective function for any integer feasible solution.

Let's branch!¶

Before implementing the branch and bound method, we solve the LP relaxation for every subproblem where we gradually increase the number of integrality restrictions in the original optimization problem. This exercise essentially means that we are applying the branching portion of the branch and bound method. Of course, the branch and bound method arrives at the solution much quicker and it is not necessary to look at the solution at every node. But again, we are trying to gain some insight into this type of problems, so it is instructive to look at it in steps.

First, we solve the LP relaxation subproblem where x1=0x1=0 and x2,x3,x4∈[0,1]x2,x3,x4∈[0,1]. Second, we solve the LP relaxation subproblem where x1=1x1=1 and x2,x3,x4∈[0,1]x2,x3,x4∈[0,1]. Then we solve the LP relaxation subproblem where x1=0x1=0, x2=0x2=0 and x3,x4∈[0,1]x3,x4∈[0,1], and so forth. We evaluate the LP relaxation at every node (e.g., x1=1x1=1 and x2=0x2=0). Then we show how to apply the branch and bound method which results in a smaller number of subproblems computation needed to arrive at the solution.

The code below first branches on x1x1,then on x1,x2x1,x2, then on x1,x2,x3x1,x2,x3, and finally it branches on x1,x2,x3,x4x1,x2,x3,x4. All duplicate nodes are eliminated at the last step - for example, 0,0,0,00,0,0,0 and 0,1,1,10,1,1,1 are child to the first node were x1=0x1=0, therefore it is not necessary to compute the value of the objective function at the first branching. The table below shows the results from the branching process. There are 3030 nodes, therefore, 3030 optimization problems solved. Why 30? Well, it is 2+22+23+24=302+22+23+24=30 since there are 22 nodes in the first branching, 44 in the second, 88 in the third, and 1616 in the forth.

Next, we look at the following subproblems:

  • IP(3): x1=0x1=0 and x2=0x2=0;
  • IP(4): x1=0x1=0 and x2=1x2=1;
  • IP(5): x1=1x1=1 and x2=0x2=0;
  • IP(6): x1=1x1=1 and x2=1x2=1;

The results show that IP(6) gives us a lower bound of 2727 since all solutions are integers. In other words, any other subproblem (with or without all integer solutions) must have an objective function greater than 2727. IP(6) becomes the incumbent solution and any other solution with integers only has to have a higher objective function, otherwise it must be eliminated. Since the value of the objective functions for IP(3) and IP(4) are less than 2727, we can eliminate these problems and any other subproblems branching out from them.

A Financial or Clustering Example¶

This example shows how integer programming can be applied to passive asset management. Suppose there are 66 stocks in an index fund, and we want to construct a fund with 33 stocks that best tracks a market index.

The goal is to create a fund that tracks the index as closely as possible restricted to a smaller number of stocks in the index - we will use correlation across security returns as a measure of association. One requirement is that only three out of the six stocks can be used in the fund. This is an important requirement because the transaction cost of tracking an index by buying all stocks in the index and rebalancing the portfolio over time is very high.

The second requirement is that a stock in the index can only be represented by one stock in the fund. This requirement implies that the stocks in the index are separated into groups with each group being represented in the fund by one stock within the group.

The fund objective and requirements can be represented mathematically:

minx,ys.t.∑i=1N∑i=1Nxijρij∑j=1Nyj=q∑j=1Nxij=1xij≤yj∀i,j=1,2,...,nxij∈{0,1}yj∈{0,1}minx,y∑i=1N∑i=1Nxijρijs.t.∑j=1Nyj=q∑j=1Nxij=1xij≤yj∀i,j=1,2,...,nxij∈{0,1}yj∈{0,1}

The metric ρi,jρi,j represents the correlation between stock ii and jj in the index. The control variable xijxij takes the value of 11 if selected to maximize the degree of association between the fund and the index and 00 otherwise. The choice variable yjyj takes the value of 11 if stock jj is in the fund and 00 otherwise. xij≤yjxij≤yj implies that a stock ii can be represented by a stock jj only if stock jj is in the fund.
