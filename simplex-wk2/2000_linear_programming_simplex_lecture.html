Lecture 02: Linear Programming

This lecture discusses the fundamental theorem of linear programming as well as the dual problem and the simplex method.

Fundamental Theorem

The fundamental theorem of linear programming establishes the existance of a basic optimal feasible solution to a linear programming problem as a necessary condition for the existance of an optimal feasible solution. In other words, if there is an optimal feasible solution to a linear program, then there is a basic optimal feasible solution. Intuitively, a basic solution to a system of linear constraints correspond to extreme points in the feasible region where one or more variables are zero. The fundamental theorem of linear programming is an important result because a linear program will not have a solution unless it has a basic or corner solution. The fundamental theorem of linear programming becomes an important tools since we are interested not only in a solution that satisfies a set of linear constraints, but also a set of nonnegative constraints (feasibility) and that optimizes the objective function in a linear programming problem (optimality).

Matrix A

As discussed in the previous lecture, a linear program can be written in matrix notation as:

minxcTxAxx=b0minxcTxAx=bx0

where xx and cc are n1n1 vectors, AA is a mnmn matrix, bb is a m1m1 vector. The inequality x0x0 means that each component of xx is nonnegative. To avoid trivial complications, we restrict attention to a subset of structures for the matrix AA. We assume that AA has full rank which means that:

  • the number of variables in xx is greater than the number of equations or linear constraints (n>mn>m).

  • linear independence across the mm equations

Basic Solution

Intuitively, a basic solution to a system of linear constraints correspond to extreme points in the feasible region where one or more variables are zero. This section formally defines a basic solution. Suppose we have a system of linear equations Ax=bAx=b and that AA has full rank. It can be shown that if AA has full rank, then the system Ax=bAx=b has at least a solution. Next, we rearrange the columns of AA in such a way that the first mm columns are linearly independent:

A[BK]A[BK]

where BB is a mmmm basis matrix formed by mm linearly indepedent columns. Similarly, we rearrange xx as:

x[xBxK]x[xBxK]

The constraints for the linear programming problem can be rewritten as:

[BK][xBxK]=b[BK][xBxK]=b

Multiplying out and rearranging

BxB+KxK=bBxB+KxK=b

Multiplying each sides by B1B1 we get:

xB+B1KxK=B1bxB+B1KxK=B1b

Solving for xBxB:

xB=B1bB1KxKxB=B1bB1KxK

Notice that xBxB is written as a function of xKxK. If (xB,xK)(xB,xK) solves Ax=bAx=b, then it is called a solution. If in addition (xB,xK)0(xB,xK)0, then it called a feasible solution. If in addition (xB,xK)(xB,xK) optimizes the objective function in the linear programming problem then it is called an optimal feasible solution. Lastly, a solution is considered a basic solution if (xB,xK)=(B1b,0)(xB,xK)=(B1b,0) which corresponds to extreme points in the feasible region.

Now we are ready to state the fundamental theorem of linear programming:

  • If there is a feasible solution (xB,xK)=(B1bB1KxK,xK)(xB,xK)=(B1bB1KxK,xK), then there is a basic feasible solution (xB,xK)=(B1b,0)(xB,xK)=(B1b,0).

  • If there is a optimal feasible solution (xB,xK)=(B1bB1KxK,xK)(xB,xK)=(B1bB1KxK,xK), then there is a basic optimal feasible solution (xB,xK)=(B1b,0)(xB,xK)=(B1b,0).

A proof of the fundamental theorem of linear programming can be found in Luenberger-Ye (Ch. 2). In words, when a linear programming problem has a solution, it must also have a basic solution. This result is important because it narrows the search for an optimal solution to corners or extreme points only, effectively reducing the domain to a finite and countable set. In fact, if there are mm equations and nn variables, there are n!m!(nm)!n!m!(nm)! basic solutions.

Search Method

The fundamental theorem of linear algebra narrows the search for an optimal solution within the feasible region to a finite and countable set of basic solutions. If the linear program has mm equations and nn variables, then there are n!m!(nm)!n!m!(nm)! basic solutions. We can then evaluate the possible solutions and choose the basic solution with the highest or lowest value for the objective function. This method for searching a solution is inefficient and time consuming, but it works!

Let's go over an example. Suppose we have the following linear programming problem:

Simplex Method

This section introduces the simplex method which relies on the fundamental theorem of linear programming. As discussed in the previous section, the results from the fundamental theorem of linear programming yield a finite search method for the optimal feasible basic solution to a linear programming problem. This finite search method is, however, inefficient. As the number of equations and variables increase, the time needed to find an optimal feasiable basic solution increases substantially. The simplex method builds upon this finite search method and yield a more efficient search procedure.

Recall that the basic variables can be expressed as a function of the non-basic variables.

xB=B1bB1KxKxB=B1bB1KxK

Similarly, we can express the objective function in terms of the non-basic variables, xKxK:

z=cTx=cTBxB+cTKxK=cTB(B1bB1KxK)+cTKxK=cTBB1b+(cTKcTBB1K)xKz=cTx=cBTxB+cKTxK=cBT(B1bB1KxK)+cKTxK=cBTB1b+(cKTcBTB1K)xK

Notice that =cTKcTBB1K=cKTcBTB1K is a vector of coefficients. If all the coefficients in are greater than or equal to zero, then the existing basic solution (xB,0)(xB,0) is optimal. That is the case because increasing any of the non-basic variables in xKxK will only increase or keep the value of the objective function unchanged. If one or more coefficients in are less than zero, then the existing basic solution is not optimal since the objective function zz can be reduced by increasing the value of one or more non-basic variables in xKxK.

Let NKNK and NBNB denote the sets containing the indices of the columns in KK and BB, respectively. Suppose i<0i<0 for some iNKiNK and ijij for every jiNKjiNK. Further, define TKA()TKA() as a function mapping a column number in KK to a corresponding column number in AA. In this context, xTKA(i)xTKA(i) is a good candidate among the non-basic variables to be increased. The next task is to determine how much we can increase xTKA(i)xTKA(i). To increase xTKA(i)xTKA(i), we must reduce the value of a non-basic variable without violating the linear restrictions. Increasing the value of one of the non-basic variables effectively enters this variable into the set of basic variables (or basis). We want to increase the value of this non-basic variable, xTKA(i)xTKA(i), as much as possible, but without violating the constraints. Recall again that the basic variables can be expressed as a function of the non-basic variables:

xBxB=bB1KxK=byTKA(i)xTKA(i)xB=bB1KxKxB=byTKA(i)xTKA(i)

where yTKA(i)yTKA(i) is the TKA(i)TKA(i)th column in the matrix AA and xTKA(i)xTKA(i) is a scalar and the TKA(i)TKA(i)th variable in xx. To find the highest value for xTKA(i)xTKA(i) that does not violate the contraints, we set xBxB equal to zero and perform an element-by-element division:

jargmin{bjyj,TKA(i):yj,TKA(i)>0}jNBjargmin{bjyj,TKA(i):yj,TKA(i)>0}jNB

The basic variable xjxj represents the basic variable that should exist the basis and take the value of 00. The bjyj,TKA(i)bjyj,TKA(i) represents the value that the variable xT(i)xT(i), entering the basis, should take.

Example

Next, we apply the simplex method to solve the optimization problem in the beginning of this section. Recall that the search method solved the optimization problem by evaluating the objective function at each corner or basic solution. The optimal basic solution was the basic solution associated with the lowest objective function value. The search method requires the evaluation of the objective function for every basic solution. As the number of equations and variables increase, the number of basic solutions increases and so does the time needed to find the optimal basic solution.

From a initial set of basic and non-basic variables, the simplex method shows how to improve the value of the objective function by swapping variables between the set of basic and non-basic variables. As the example will show, the number of iterations necessary to attain the optimal solution is significant less than under the naive search method.

Before we assign the values for xBxB and xKxK, we must create [BK][BK] from AA. Step 2 - Determine non-basic variable to enter the basis

In this step, we first determine if increasing the value of a non-basic variable from zero decreases the value of the objective function. In the previous section, we learned that the objective function can improve if one or more coefficients in =cTKcTBB1K=cKTcBTB1K are greater than zero. We must determine whether the objective function can be further minimized or not by reviewing the sign of the coefficients of =cTKcTBB1K=cKTcBTB1K which represents the change in the value of th objective function due to an unit change in the value each of the non-basic variables. If i<0i<0 for some ii, then it is possible to reduce the value of the objective function by increase the value of one of the non-basic variables.

Once we identified one or more coefficients in are greater than zero, we can then determine which non-basic variable should enter the basis. A non-basic variable should enter the basis if increasing its value from zero decreases the current value of the ojective function, zz. In the case of multiple non-basic variables with negative coefficients, we should pick the variable that improves the objective function the most on a per unit basis.

In the current example, the objective function can be further minimized by increasing the value of x1x1 and/or x2x2. Since there are two non-basic variables to choose from, we should pick the one that can reduce the objective function the most per unit increase. Base on this rationale, x1x1 should be choosen since a increase in x1x1 decreases the objective function by more than the same increase in x2x2. The non-basic variable x1x1 is then choosen to enter the basis.

Step 3 - Determine basic variable to exit the basis

By increasing x1x1 from its original value of 00, we effectively convert x1x1 to a basic variable. The next step is to determine which basic variable, xBxB, to exit the basis. A basic variable exits the basis and become a non-basic variable when its value changes to 00. In order to convert a basic variable to a non-basic variable, we must increase the value of x1x1 and decrease the value of the basic variables to 00; but we must do so without violating the linear contraints.

The ratio test, described in the previous section, determines which basic variable to exist the basis given the non-basic variable entering the basis (i.e., x1x1).
